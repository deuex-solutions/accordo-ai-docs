## Fix 1: A1 — Conversation Arc Summary (RICE: 6 but Highest Potential Impact)

### Problem
The persona-renderer currently sends the LLM ONLY the latest vendor message. The LLM has zero visibility into the negotiation history — it doesn't know:
- What price the vendor started at or how far they've moved
- What the PM said in previous rounds
- How many rounds have passed (beyond a round number)
- What the trajectory of the negotiation looks like

This is the #1 root cause of generic, context-free responses. A human PM negotiating round 7 would sound completely different from round 1, referencing the journey, the progress made, the relationship built. The current system sounds like round 1 every time.

### Design Constraints
- **Hard boundary**: The arc summary must NOT include utility scores, weights, thresholds, target prices, max prices, config objects, or any internal scoring data
- **Token budget**: The summary must be compact — the LLM instruction is already dense. Target ~100–150 words max for the arc
- **Deterministic**: The arc is built from message history (DB data), not generated by the LLM
- **Safe fields only**: Vendor prices, PM counter prices (already exposed via `allowedPrice`), payment terms, and PM actions (COUNTER/ACCEPT/etc.) — these are vendor-visible information, not internal strategy

```typescript
/**
 * Arc Summary Builder
 *
 * Builds a compact, deterministic summary of the negotiation arc from message
 * history. Passed to the persona-renderer so the LLM can produce contextually
 * aware responses that reference the journey.
 *
 * Hard boundary rules:
 * - ONLY uses vendor-visible information (prices both parties stated, terms, actions)
 * - NEVER includes: utility scores, weights, thresholds, target/max prices, config
 * - Deterministic: pure function from message history, no LLM involved
 * - Compact: targets ~100–150 words for the entire arc string
 */

import type { ChatbotMessage } from "../models/chatbot-message.js";

export interface ArcRound {
  round: number;
  vendorPrice: number | null;
  vendorTerms: string | null;
  pmAction: string | null;
  pmPrice: number | null;
  pmTerms: string | null;
}

/**
 * Extract structured round-by-round data from message history.
 * Messages are paired: VENDOR message → ACCORDO response = one round.
 */
export function extractArcRounds(
  messages: Array<{
    role: string;
    content: string;
    extractedOffer?: any;
    counterOffer?: any;
    decisionAction?: string | null;
  }>,
): ArcRound[] {
  const rounds: ArcRound[] = [];
  let currentRound: Partial<ArcRound> = { round: 1 };

  for (const msg of messages) {
    if (msg.role === "VENDOR") {
      // Start a new round with vendor data
      const offer = msg.extractedOffer as any;
      currentRound.vendorPrice = offer?.total_price ?? null;
      currentRound.vendorTerms = offer?.payment_terms ?? null;
    } else if (msg.role === "ACCORDO" && msg.decisionAction) {
      // Complete the round with PM response
      const counter = msg.counterOffer as any;
      currentRound.pmAction = msg.decisionAction;
      currentRound.pmPrice = counter?.total_price ?? null;
      currentRound.pmTerms = counter?.payment_terms ?? null;

      rounds.push({
        round: currentRound.round ?? rounds.length + 1,
        vendorPrice: currentRound.vendorPrice ?? null,
        vendorTerms: currentRound.vendorTerms ?? null,
        pmAction: currentRound.pmAction,
        pmPrice: currentRound.pmPrice ?? null,
        pmTerms: currentRound.pmTerms ?? null,
      });

      currentRound = { round: rounds.length + 1 };
    }
  }

  return rounds;
}

/**
 * Build a compact, human-readable negotiation arc summary.
 * 
 * Format: "Round 1: Vendor offered $X, you countered $Y. Round 2: ..."
 * Only includes rounds where prices were exchanged.
 * Keeps to ~100–150 words even for long negotiations by summarizing early rounds
 * and detailing recent ones.
 *
 * @param messages - All ChatbotMessages for this deal, in chronological order
 * @param currencySymbol - Currency symbol for formatting (e.g. "$", "£")
 * @returns Arc summary string, or empty string if insufficient history
 */
export function buildArcSummary(
  messages: Array<{
    role: string;
    content: string;
    extractedOffer?: any;
    counterOffer?: any;
    decisionAction?: string | null;
  }>,
  currencySymbol: string = "$",
): string {
  const rounds = extractArcRounds(messages);

  // Need at least 2 rounds for an arc to be meaningful
  if (rounds.length < 2) return "";

  const formatPrice = (price: number | null): string =>
    price != null ? `${currencySymbol}${price.toLocaleString("en-US")}` : "no price stated";

  const lines: string[] = [];

  // For negotiations with > 5 rounds, summarize early rounds and detail recent ones
  if (rounds.length > 5) {
    // Summarize rounds 1 through N-3
    const earlyRounds = rounds.slice(0, rounds.length - 3);
    const firstVendorPrice = earlyRounds.find((r) => r.vendorPrice != null)?.vendorPrice;
    const lastEarlyVendorPrice = [...earlyRounds].reverse().find((r) => r.vendorPrice != null)?.vendorPrice;
    
    lines.push(
      `Rounds 1–${earlyRounds.length}: Vendor started at ${formatPrice(firstVendorPrice)}` +
        (lastEarlyVendorPrice != null && lastEarlyVendorPrice !== firstVendorPrice
          ? ` and moved to ${formatPrice(lastEarlyVendorPrice)}`
          : "") +
        `. ${earlyRounds.filter((r) => r.pmAction === "COUNTER").length} counters exchanged.`,
    );

    // Detail the last 3 rounds
    const recentRounds = rounds.slice(rounds.length - 3);
    for (const r of recentRounds) {
      const vendorPart = r.vendorPrice != null ? `Vendor: ${formatPrice(r.vendorPrice)}` : "Vendor restated";
      const pmPart =
        r.pmAction === "COUNTER" && r.pmPrice != null
          ? `you countered ${formatPrice(r.pmPrice)}`
          : r.pmAction
            ? `you ${r.pmAction.toLowerCase().replace("_", " ")}ed`
            : "";
      const termsPart = r.vendorTerms ? ` (${r.vendorTerms})` : "";
      lines.push(`Round ${r.round}: ${vendorPart}${termsPart}, ${pmPart}.`);
    }
  } else {
    // 2–5 rounds: detail each one
    for (const r of rounds) {
      const vendorPart = r.vendorPrice != null ? `Vendor offered ${formatPrice(r.vendorPrice)}` : "Vendor message (no price)";
      const pmPart =
        r.pmAction === "COUNTER" && r.pmPrice != null
          ? `you countered at ${formatPrice(r.pmPrice)}`
          : r.pmAction
            ? `you ${r.pmAction.toLowerCase().replace("_", " ")}ed`
            : "you responded";
      const termsPart = r.vendorTerms ? ` (${r.vendorTerms})` : "";
      lines.push(`Round ${r.round}: ${vendorPart}${termsPart}, ${pmPart}.`);
    }
  }

  return `Negotiation history (${rounds.length} rounds so far):\n${lines.join("\n")}`;
}
```

## Fix 2: L2 — Lock `toLocaleString` to Consistent Locale

### Problem
Every call to `allowedPrice?.toLocaleString()` in the fallback templates and persona-renderer uses the **server's default locale**. In production, Node.js locale depends on the OS configuration of the server instance. This means:
- US server: `98500` → `"98,500"` ✓
- German server: `98500` → `"98.500"` (period as thousands separator!)
- Indian server: `98500` → `"98,500"` but `31500000` → `"3,15,00,000"` (lakh grouping)

The price validator expects US-style comma-separated numbers. A German-locale server would format `$98.500` which the validator's regex wouldn't match correctly (the period looks like a decimal), causing false validation failures.

### Solution
Replace all bare `.toLocaleString()` calls with `.toLocaleString("en-US")` to lock to US-English number formatting (commas for thousands, period for decimal). This is consistent across all deployment regions.
